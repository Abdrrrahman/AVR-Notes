### **Note 3:**
Any incrementing or decrementing on a variable and using the same variable in the same statement is an undefined behavior, You have to make these operation 
**Undefined Behavior**
```c
#include <stdio.h>

int main () {
	unsigned char x = 5;
	printf("%i\t%i\t%i\t", ++x, ++x, ++x);
	return 0;
}
```
**Undefined Behavior**
```c
#include <stdio.h>

int main () {
	unsigned char x = 5;
	printf("%i", x++ +x);
	return 0;
}
```
**Valid**
```c
#include <stdio.h>

int main () {
	unsigned char x = 5, y=2;
	printf("%i", x++ +y);
	return 0;
}
```
### **Note 4:**
#### Understanding Lvalue vs Rvalue:
- **Lvalue (Left value)**: An expression that refers to a **memory location**, i.e., it has an **address**. You can assign a value to it.
    - Example: `x = 5;` → `x` is an lvalue.
- **Rvalue (Right value)**: A **value** that does **not have a memory address**, i.e., it's a **temporary result** or constant.
    - Example: `x = y + 2;` → `y + 2` is an rvalue.  

```c
#include <stdio.h>
int main() {
	int x=1;
	++(x++); }
```
#### What happens here:
- `x++` is a **post-increment**, which:
    - Returns the **current value of x (1)** → This is an **rvalue**.    
    - **Increments x** after the value is used → `x` becomes 2.    
- `++(x++)` tries to apply **pre-increment** (`++`) to the **result of `x++`**, which is an rvalue (a temporary value, not a variable with an address).
##### Error:
You **cannot apply `++` to an rvalue**, because `++` needs to **modify the actual memory** (needs an lvalue).

### **Note 11:**
Bitwise Operators: 
Makes the bitwise logic (and, or, not, xor, ..) on each bit, unlike the logical operators that check the whole logic.
```c
#include <stdio.h>
int main() {
	int a = 1, b = 4, bitwise_result, logical_result;
    logical_result = a && b;
    bitwise_result = a & b;
    printf("%i", logical_result); // output is -> 1
    printf("%i", bitwise_result); // output is -> 0
    return 0; }
```

### **Note 12:**
#### **Shift left (<<):**
It inserts a zero in the least significant bit -- right most bit --  then the rest of the bits are shifted left, this means that the number is doubled due to the decimal binary system, so shifting left is like multiplying the number by 2.
![[Pasted image 20250718161244.png]] 
So shifting the number by n, the result is the number * 2^n
```c
#include <stdio.h>
int main() {
    unsigned char x=15, z;
    z=x<<2; //The X here does not change
    printf("%i\n",x);
    printf("%i\n",z);
    return 0; }
    
--------------------------------------------------------
//-> 15
//-> 60
```

```c
#include <stdio.h>
int main() {
    unsigned char x=64;
    unsigned short z=64;
    x<<=2; 
    // two bits has no space to be hold in as char is only 8 bits and there is no space to hold the rest bits
    z<<=2;
    printf("%i\n",x);
    printf("%i\n",z);
    return 0; }
    
--------------------------------------------------------
//-> 0
//-> 256
```

###### **The shift left has no special case.**
#### **Shift Right(>>):**
It inserts a zero in the most significant bit -- left most bit --  then the rest of the bits are shifted right, this means that the number is divided by 2 due to the decimal binary system, so shifting right is like dividing the number by 2.
![[Pasted image 20250718162530.png]]
So shifting the number by n, the result is the number / 2^n

##### **The shift right has a special case with dealing with signed data types.**
If the variable is negative, then the most significant bit is 1, so it inserts ones instead of zeros when dealing with negative signed, so it keeps the two's complement valid.

```c
#include <stdio.h>
int main() {
    signed char x=63;
    x>>=2;
    printf("x=%i\n",x);
    return 0; }
    
-----------------------------------------------------
//-> x=15
```

```c
#include <stdio.h>
int main() {
    signed char x=-6;
    x>>=2;
    printf("x=%i\n",x);
    return 0; }
-----------------------------------------------------
//-> x=-2
```
