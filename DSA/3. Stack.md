# Stack Overview
Stack is a FILO/LIFO data structure: the last element you stored is the first element you retrieve, or the first element you stored is the last element you retrieve. It is the same as the pile-of-dishes analogy, where elements can be added or removed only at one side, maybe the top or the bottom (implementation defined), but we will standardize our article here that we add/remove only at the top.

Storing in a stack is named `push`, and retrieving from it is named `pop`. Stack has a stack pointer, which is a pointer pointing to the memory address the stack is currently on (the place that data will be pushed to or popped from).

In the beginning, the stack pointer is pointing to the stack bottom, which means that the stack is empty. If you called `push`, then the stack pointer will move one cell, and if you called `pop`, then the stack pointer will move backward one cell so it will point to its bottom again.

You don't need to delete the data in the popped cells; it will be overridden at the next push.

The implementation of the `push` method is really intuitive: you just increment the stack pointer one step, then you enter your data. And the implementation of `pop` is intuitive too: you just decrement the stack pointer one step. In some implementations, `pop` returns the popped value, so you need to store the data before you pop it. Another method is `peek`, which returns the value of the top of the stack without removing the element.

Stack has two implementations:
- Linked List Stack
- Array Stack

We'll implement both.
# Array Implementation
## ArrayStack.h
```c
// Code Snippet 1-1

#pragma once
#include "genericPrint.h"

#define SIZE 5

typedef struct {
    void *data;
    Datatype type;
} StackData;

typedef struct {
    StackData elements[SIZE];
    int topIdx;
} Stack;

void initializeStack(Stack *stack);

void push(Stack *stack, void *data, Datatype type);
StackData pop(Stack *stack);
StackData peek(const Stack *stack);

int isFull(const Stack *stack);
int isEmpty(const Stack *stack);

void displayElements(const Stack *stack);
int numberOfElements(const Stack *stack);
void destroyStack(Stack *stack);
```

This is the header file of an array-based stack. `SIZE` (defined as 5 here) sets the stack's fixed capacity, since the array is allocated at a fixed length, this is the maximum number of elements the stack can ever hold.

Here we want each cell in the stack to contain both the data it holds and an enum value representing the datatype of the data, to be used by `genericPrint`.

So, we created the first struct named `StackData`, which has two members: `data` and `datatype`.
```c
// Code Snippet 1-2

typedef struct {
	void *data;
	Datatype type;
} StackData;
```

Then we created another struct, which is the stack itself, which contains an array of `StackData` elements and another member defining the stack pointer.
```c
// Code Snippet 1-3

typedef struct {
	StackData elements[SIZE];
	int topIdx;
} Stack;
```

Then we have the prototypes of each method; each method will be explained in its implementation section below.
## ArrayStack.c
```c
// Code Snippet 1-4

#include "ArrayStack.h"
#include <stdio.h>

void initializeStack(Stack *stack) {
	stack->topIdx = -1;
}

void push(Stack *stack, void *data, Datatype type) {
	if(isFull(stack)) {
		fprintf(stderr, "Stack Overflow!\n");
	} else  {
		stack->elements[++stack->topIdx] = (StackData){data, type}; 
	}
}

StackData pop(Stack *stack) {
	StackData returnData = {nullptr, 0};
	
	if(isEmpty(stack)) {
		fprintf(stderr, "Can not pop from an empty stack!\n");
	} else {
		returnData = stack->elements[stack->topIdx--];
	}
	
	return returnData;
}

StackData peek(const Stack *stack) {
	StackData returnData = {nullptr, 0};
	
	if(isEmpty(stack)) {
		fprintf(stderr, "Stack is empty!\n");
	} else {
		returnData = stack->elements[stack->topIdx];
	}
	
	return returnData;
}

int isFull(const Stack *stack) {
	return (stack->topIdx == SIZE - 1);
}

int isEmpty(const Stack *stack) {
	return (stack->topIdx == -1);
}

void displayElements(const Stack *stack) {
	if(isEmpty(stack)) {
		fprintf(stderr, "Can not display an empty stack!\n");
	} else {
		for (int i = stack->topIdx; i >= 0; i--) {  
			if(stack->topIdx == i) {
				printf("Top ->\t");
			} else {
				printf("\t\t");
			}
			genericPrint(stack->elements[i].data);
		}
	}
}

int numberOfElements(const Stack *stack) {
	return (stack->topIdx + 1);
}

void destroyStack(Stack *stack) {
	initializeStack(stack);
}
```

### `void initializeStack(Stack *stack);`
This method initializes the stack by setting its top index to -1. This is done here because we can't set a global variable named `topIdx` for all struct objects and set it to -1 — it would be shared among all objects.

```c
// Code Snippet 1-5

void initializeStack(Stack *stack) {
	stack->topIdx = -1;
}
```

### `void push(Stack *stack, void *data, Datatype type);`
This method enters the data given to it into the current top index, where the stack pointer is. First it checks if the stack is not full: if not, it increments `topIdx` by 1 — as it is currently pointing to the last pushed element — then it inserts a new stack node containing the data and type entered to the method at the new `topIdx`.

```c
// Code Snippet 1-6

void push(Stack *stack, void *data, Datatype type) {
	if(isFull(stack)) {
		fprintf(stderr, "Stack Overflow!\n");
	} else  {
		stack->elements[++stack->topIdx] = (StackData){data, type}; 
	}
}
```

### `StackData pop(Stack *stack);`
This method returns the node at the top index, then removes it from the stack. First, it creates a new stack node named `returnData` to hold the value to be returned by the method. `returnData` should hold a default value if the method fails and the top node otherwise. To do so, first it checks whether the stack is empty or not: if not, then it assigns `returnData` to the top index, then it decrements the top index.

```c
// Code Snippet 1-7

StackData pop(Stack *stack) {
	StackData returnData = {nullptr, 0};
	
	if(isEmpty(stack)) {
		fprintf(stderr, "Can not pop from an empty stack!\n");
	} else {
		returnData = stack->elements[stack->topIdx--];
	}
	
	return returnData;
}
```

### `StackData peek(const Stack *stack);`
This method returns the top node only and does nothing to the actual stack, so it does the same as `pop` except for changing the stack pointer (top index). It receives the stack as `const`, meaning that it can not change anything in it (read only).

```c
// Code Snippet 1-8

StackData peek(const Stack *stack) {
	StackData returnData = {nullptr, 0};
	
	if(isEmpty(stack)) {
		fprintf(stderr, "Stack is empty!\n");
	} else {
		returnData = stack->elements[stack->topIdx];
	}
	
	return returnData;
}
```

### `int isFull(const Stack *stack);`
This method is a boolean method that returns 1 (true) when the stack is full and 0 (false) otherwise. We use `int` instead of `bool`, as C doesn't have a `bool` datatype.

```c
// Code Snippet 1-9

int isFull(const Stack *stack) {
	return (stack->topIdx == SIZE - 1);
}
```

### `int isEmpty(const Stack *stack);`
This method is a boolean method that returns 1 (true) when the stack is empty and 0 (false) otherwise. We use `int` instead of `bool`, as C doesn't have a `bool` datatype.

```c
// Code Snippet 1-10

int isEmpty(const Stack *stack) {
	return (stack->topIdx == -1);
}
```

### `void displayElements(const Stack *stack);`
This method displays the data of each node in the stack.

```c
// Code Snippet 1-11

void displayElements(const Stack *stack) {
	if(isEmpty(stack)) {
		fprintf(stderr, "Can not display an empty stack!\n");
	} else {
		for (int i = stack->topIdx; i >= 0; i--) {  
			if(stack->topIdx == i) {
				printf("Top ->\t");
			} else {
				printf("\t\t");
			}
			genericPrint(stack->elements[i].data);
		}
	}
}
```

### `int numberOfElements(const Stack *stack);`
This method returns the number of elements in a stack.

```c
// Code Snippet 1-12

int numberOfElements(const Stack *stack) {
	return (stack->topIdx + 1);
}
```

### `void destroyStack(Stack *stack);`
Unlike the linked list implementation, an array-based stack does **not** allocate memory dynamically. All elements are stored inside a fixed-size array that is allocated when the stack itself is created.

Therefore, there is no dynamically allocated memory to deallocate using `free()`.
Destroying the stack simply means resetting it to its initial empty state by calling `initializeStack()`.

This resets the front index, rear index, and size, effectively removing all elements from the queue.
```c
// Code Snippet 2-16

void destroyStack(Stack *stack) {
	initializeStack(stack);
}
```

# Linked List Implementation
## LinkedStack.h

```c
// Code Snippet 2-1

#pragma once

#include "genericPrint.h"

typedef struct {
	void *data;
	Datatype type;
} StackData;

typedef struct StackNode {
	StackData compositeData; 
	struct StackNode *next;
} StackNode;

typedef struct {
	StackNode *topNode;
	int size;
} Stack;

void initializeStack(Stack *stack);  
  
void push(Stack *stack, void *data, Datatype type);  
StackData pop(Stack *stack);  
StackData peek(const Stack *stack);  
  
int isEmpty(const Stack *stack);  
int numberOfElements(const Stack *stack);  
  
void displayElements(const Stack *stack);  
void destroyStack(Stack *stack);
```

In the array-based stack we had a `StackData` struct containing data and datatype. In the linked list, we will have the same, in addition to a next pointer that points to the following node.

In the array-based stack we had a `Stack` struct that contains a stack pointer that points to the top index. Here, we have a stack pointer that points to the top node, in addition to a size variable that keeps track of the size of the stack.

We don't have an `isFull()` method here, as the linked list is growable, it never gets full.

We have a new method here called `destroyStack()`; similar to `destroyList()`, it destroys (deallocates) every allocated node on the heap.

## LinkedStack.c
```c
// Code Snippet 2-2

#include <stdio.h>
#include <stdlib.h>
#include "LinkedStack.h"

void initializeStack(Stack *stack) {
	stack->topNode = nullptr;
	stack->size = 0;
}

void push(Stack *stack, void *data, Datatype type) {
	StackNode *newNode = malloc(sizeof(StackNode));
	if(newNode == nullptr) {
		fprintf(stderr, "Memory Allocation Failed!\n");
	} else {
		newNode->compositeData.data = data;
		newNode->compositeData.type = type;
		newNode->next = stack->topNode;
		
		stack->topNode = newNode;
		stack->size++;
	}
}

StackData pop(Stack *stack) {
	StackData returnData = {nullptr, 0};
	
	if(isEmpty(stack)) {
		fprintf(stderr, "Can not pop an empty stack!\n");
	} else {
		StackNode *freeMe = stack->topNode;
		returnData = freeMe->compositeData;
		
		stack->topNode = freeMe->next;
		stack->size--;
		
		free(freeMe);
	}
	
	return returnData;
}

StackData peek(const Stack *stack) {
	StackData returnData = {nullptr, 0};
	
	if(isEmpty(stack)) {
		fprintf(stderr, "Can not peek an empty stack!\n");
	} else {
		returnData = stack->topNode->compositeData;
	}
	
	return returnData;
}

int isEmpty(const Stack *stack) {
	return (stack->topNode == nullptr);
}

int numberOfElements(const Stack *stack) {
	return stack->size;
}

void displayElements(const Stack *stack) {
	if(isEmpty(stack)) {
		fprintf(stderr, "Can not display an empty stack!\n");
	} else {
		StackNode *current = stack->topNode;
		while(current != nullptr) {
			if(current == stack->topNode) {
				printf("Top ->\t");
			} else {
				printf("\t\t");
			}
			genericPrint(current->compositeData.data);
			current = current->next;
		}
	}
}

void destroyStack(Stack *stack) {
	while(!isEmpty(stack)) {
		pop(stack);
	}
}
```

The linked list implementation of the stack is the same as a normal linked list, but you push and pop at one end (head or tail); we will choose the head to reduce complexity.

### `void initializeStack(Stack *stack);`
This method initializes the struct's members into their default values.

```c
// Code Snippet 2-3

void initializeStack(Stack *stack) {
	stack->topNode = nullptr;
	stack->size = 0;
}
```

### `void push(Stack *stack, void *data, Datatype type);`
This method pushes a node containing the data and type entered to the head of the linked list. First it allocates a `newNode`, then it checks if it is not `nullptr` — allocation succeeded — then it assigns `data` and `type` with their values, and `next` of the `newNode` to the `topNode` — head — then it makes `topNode` the newly entered node (`newNode`), then increments size.

```c
// Code Snippet 2-4

void push(Stack *stack, void *data, Datatype type) {
	StackNode *newNode = malloc(sizeof(StackNode));
	
	if(newNode == nullptr) {
		fprintf(stderr, "Memory Allocation Failed!\n");
	} else {
		newNode->compositeData.data = data;
		newNode->compositeData.type = type;
		newNode->next = stack->topNode;
		
		stack->topNode = newNode;
		stack->size++;
	}
}
```

### `StackData pop(Stack *stack);`
This method returns `StackData`, which is a composite datatype of two pieces: data and type.

First it creates an object of `StackData` to be the return value, named `returnData`, and initializes it with the default value for now. Then it checks if the stack is not empty: if not, then it creates a new `StackNode` named `freeMe` that points to the stack's `topNode`, then it makes `returnData` contain the `compositeData` of the `topNode`, then it advances `topNode` to point to the next node and decrements `size`, then it frees `freeMe` and returns `returnData`.

```c
// Code Snippet 2-5

StackData pop(Stack *stack) {
	StackData returnData = {nullptr, 0};
	
	if(isEmpty(stack)) {
		fprintf(stderr, "Can not pop an empty stack!\n");
	} else {
		StackNode *freeMe = stack->topNode;
		returnData = freeMe->compositeData;
		
		stack->topNode = freeMe->next;
		stack->size--;
		
		free(freeMe);
	}
	
	return returnData;
}
```

### `StackData peek(const Stack *stack);`
This method does the same as the one above, except it doesn't remove the `topNode`, so it doesn't change anything in the stack. It's a read-only method.

```c
// Code Snippet 2-6

StackData peek(const Stack *stack) {
	StackData returnData = {nullptr, 0};
	
	if(isEmpty(stack)) {
		fprintf(stderr, "Can not peek an empty stack!\n");
	} else {
		returnData = stack->topNode->compositeData;
	}
	
	return returnData;
}
```

This method returns whether the stack is empty or not.

### `void displayElements(const Stack *stack);`
This method displays the data in each node from beginning to end. First it checks if the stack is not empty: if so, it creates a `StackNode` named `currentNode` to loop over the nodes of the stack, looping through them one by one and printing the data member of each.

```c
// Code Snippet 2-9

void displayElements(const Stack *stack) {
	if(isEmpty(stack)) {
		fprintf(stderr, "Can not display an empty stack!\n");
	} else {
		StackNode *currentNode = stack->topNode;
		while(currentNode != nullptr) {
			if(currentNode == stack->topNode) {
				printf("Top ->\t");
			} else {
				printf("\t\t");
			}
			genericPrint(currentNode->compositeData.data);
			currentNode = currentNode->next;
		}
	}
}
```

### `void destroyStack(Stack *stack);`
This method acts like a destructor in OOP - it loops over all nodes and frees them by calling `pop`.

```c
// Code Snippet 2-10

void destroyStack(Stack *stack) {
	while(!isEmpty(stack)) {
		pop(stack);
	}
}
```
