A linked list is a linear collection of nodes connected by pointer links, accessed via a pointer to the first node of the list. The link pointer in the last node is set to null to mark the list's end.

|**Features**|**Array**|**Linked List**|
|---|---|---|
|**Strength**|Random Access<br>Less memory needed per element|Dynamic size<br>Faster in insertion and deletion|
|**Weakness**|Fixed size<br>Slower in insertion and deletion|Sequential access<br>More memory needed per element|

We will need a generic print function for this.

### `print_list`

`void print_list(Node *current);` It prints the data of each node in the linked list.

### `push_head`

`void push_head(Node **head, void *data, Datatype datatype);` It pushes a new node containing the data given to the head of the linked list.

### `push_tail`

`void push_tail(Node *current, Node **head, void *data, Datatype datatype);` It pushes a new node containing the data given to the end of the linked list.

### `pop_head`

`void* pop_head(Node **head);` It returns the head of the linked list, then removes it and assigns the head pointer to the next node.

### `pop_tail`

`void* pop_tail(Node *current, Node **head);` It returns the tail of the linked list, then removes it and assigns the tail pointer to the preceding node.

### `insert_by_index`

`void insert_by_index(Node *current, Node **head, void *data, Datatype datatype, int idx);` It inserts a new node containing the data given into the linked list at a specific index given.

### `remove_by_index`

`int remove_by_index(Node **head, int idx);` It removes a specific node at a specific index from the linked list.

## LinkedList.c

### `print_list`

```c
// Code Snippet 2

void print_list(Node *current) {
	
	if(current == nullptr) {
		fprintf(stderr, "Can not print an empty list!\n");
	} else {
		
		while(current != nullptr) {
			genericPrint(current->data);
			current = current->next;
		}
		
	}
}

// Usage inside main
int main(void) {
	Node* myList = nullptr;
	print_list(myList);
	
	return 0;
}
```

In `main.c`, you will have the linked list stored as a head pointer to its very first element, so we can say that head is the linked list. The function here receives the linked list (head) inside a pointer-to-node variable named `current` as well. If the head is not null, then the linked list is not empty, so I can go on.

It starts by looping over all the nodes. At each node it prints the data. The loop will stop only when you have finished the linked list, which means it will only stop when you have reached null. The data can be `int`, `short`, `double`, `char`, etc., and this creates a problem with the format specifiers in the `printf` function, so one solution is using a user-defined function that is overloaded with each `printf` of each type. To move to the next node, you have to make the `current` pointer point to the next node.

### `push_head`

```c
// Code Snippet 3

void push_head(Node **head, void *data, Datatype datatype) {
	
	Node* newNode = (Node *) malloc(sizeof(Node));
	
	if(newNode != nullptr) {
		newNode->data = data;
		newNode->datatype = datatype;
		newNode->next = *head;
		*head = newNode;
	}
}

int main(void) {
	Node* myList = nullptr;
	push_head(&myList, 10, 2);
	
	return 0;
}
```

First it creates a new node with the data given, then it makes its next pointer point to the same address the head pointer points to, then it makes the head pointer point to it.

#### Why are we passing the head as a pointer to a pointer?

Because we want to change it, and in C, everything is passed **by value** (as a copy). If you only pass a single pointer (`Node *head`), the function creates a local copy of that memory address. Any changes you make to that local pointer inside the function are completely lost when the function returns.

#### What happens if you use a single pointer (`Node *head`)?

Let's look at what goes wrong if the function signature was written like this:

```c
// Code Snippet 4

void bad_push_head(Node *head, void *data, Datatype datatype);
```

1. In `main`, you have a variable `Node *my_list = NULL;`.
2. When you call `bad_push_head(my_list, val);`, the compiler creates a **copy** of `my_list` and names it `head` inside the function.
3. Inside the function, you allocate a new node and point `head = new_node;`.
4. **The Problem:** Only the local copy `head` shifts to look at the new node. The original pointer back in `main` (`my_list`) is still sitting there pointing to `NULL`. When the function finishes, the local `head` is destroyed, and you've just leaked your new node's memory!

To fix this, you don't pass the value of the head pointer; you pass the **exact memory location where the head pointer itself lives**. Here is how the double pointer operates step-by-step:

```c
// Code Snippet 5

void push_head(Node **head, void *data, Datatype datatype) {
}
```

- **`head`**: The memory address of the pointer variable in `main`.
- **`*head`**: Dereferencing it once gives you direct, modifying access to the original pointer in `main`. Assigning `*head = new_node;` forces the pointer in `main` to permanently shift and point to your newly created head node.

### `push_tail`

```c
// Code Snippet 6

void push_tail(Node *current, Node **head, void *data, Datatype datatype) {
	
	if(current == nullptr) {
		push_head(head, data, datatype);
	} else {
		
		Node *newNode = (Node *)malloc(sizeof(Node));
		
		if(newNode != nullptr) {
			newNode->data = data;
			newNode->datatype = datatype;
			newNode->next = nullptr;
			
			while(current->next != nullptr) {
				 current = current->next;
			}
			
			current->next = newNode;
		}
	}
}

int main(void) {
	Node* myList = nullptr;
	push_head(&myList, 10, 2);
	push_tail(myList, &myList, 12, 3);
	
	return 0;
}
```

First, if the linked list's head is `nullptr` (the linked list is empty), it will delegate its task to (calls) `push_head()`. Otherwise, if the linked list is not empty, it first creates a node, then it loops until it finds the last node. Once it finds it, it makes the last node's next pointer point to the new node, and makes the new node's next pointer point to `nullptr`.

The `Node *current` is for looping, while `Node **head` is for changing the head (to be able to call `push_head()` for the case where the linked list is empty).

#### Why `current` and `head` together?

A good question to ask here is why we need `Node *current` if I can dereference `head` once to get it.

The answer is simple: I need to move a pointer through the loop, so I don't want to be changing the actual head of the linked list, which would be inconvenient. Another approach is to take a copy inside the function like this:

```c
// Code Snippet 7

void push_tail(Node **head, void *data, Datatype datatype) {
	Node* current = *head;
	
	/*
		The rest of the logic
	*/
}
```

But lol, it doesn't matter which of the two approaches you use — you created a variable to store it either way, whether inside the function's signature or inside its body.

> **The golden rule:** `Node *current` is a copy, often used for traversing. `Node **head` is the address of the head, used for editing the head position.

### `pop_head`

```c
// Code Snippet 8

void* pop_head(Node **head) {
	
	void *returnValue = nullptr;
	
	if(*head == nullptr) {
		fprintf(stderr, "Can not pop an empty linked list!\n");
	} else {
		Node *temp = *head;
		returnValue = temp->data;
		*head = (*head)->next;
		free(temp);
	}
	
	return returnValue;
}

int main(void) {
	Node* myList = nullptr;
	push_head(&myList, 19, 2);
	push_tail(myList, &myList, 38, 2);
	
	printf("%i\n", (int *) pop_head(&myList));
	printf("%i\n", (int *) pop_head(&myList));
	
	return 0;
}
```

Before anything, it checks if the linked list is not empty. If true:

1. First, it needs to make a temporary variable to store the first node (the one to be deleted).
2. Second, it needs to make the head point to the second node instead of the first, and since we need to change the head pointer, we need a pointer to a pointer to head: `Node **head`.
3. Third, it needs to return the data of the node to be deleted.
4. Fourth, it needs to deallocate the node to be deleted.

### `pop_tail`

```c
// Code Snippet 9

void* pop_tail(Node *current, Node **head) {
	
	void *returnValue = nullptr;

	if(*head == nullptr) {
		fprintf(stderr, "Can not pop an empty linked list!\n");
	} else {
		if(current->next == nullptr) {
			returnValue = pop_head(head);
		} else {
			while(current->next->next != nullptr) {
				current = current->next;
			}
			returnValue = current->next->data;
			free(current->next);
			current->next = nullptr;
		}
	}
	
	return returnValue;
}

int main(void) {
	Node* myList = nullptr;
	push_head(&myList, 19, 2);
	push_tail(myList, &myList, 38, 2);
	
	printf("%i\n", (int *) pop_tail(myList, &myList));
	printf("%i\n", (int *) pop_tail(myList, &myList));
	
	return 0;
}
```

Before anything, it checks if the linked list is not empty. If true:

1. It checks if there is only one node. If true, it needs to call `pop_head()`.
2. Otherwise, it keeps looping until the node before the last node, then it needs to make a temporary variable to store the last node's data, then makes the next pointer of the node before the last node point to `nullptr`, then deallocates the last node, then returns the temporary variable.

### `insert_by_index`

```c
// Code Snippet 10

void insert_by_index(Node *current, Node **head, void *data, Datatype datatype, int idx) {
	int counter=0;
	
	if(idx < 0) {
		fprintf(stderr, "Can not insert at an index out of range!\n");
	} else {
		if(current == nullptr || idx == 0) {
			push_head(head, data, datatype);
		} else {
			while(current!=nullptr && counter<idx-1) {
				current = current->next;
				++counter;
			}
			
			if(current == nullptr) {
				fprintf(stderr, "Can not insert at an index out of range!\n");
			} else {
				Node* newNode = (Node *) malloc(sizeof(Node));
				if(newNode != nullptr) {
					newNode->data = data;
					newNode->datatype = datatype;
					
					newNode->next = current->next;
					current->next = newNode;
				}
			}
		}
	}
}

int main(void) {
	Node* myList = nullptr;
	insert_by_index(myList, &myList, 40, 2, 0);
	insert_by_index(myList, &myList, 40, 2, 1);
	insert_by_index(myList, &myList, 40, 2, 1);
	
	return 0;
}
```

Before anything, it checks that the index is not negative. If true:

1. It checks if the linked list is empty or the index is 0. If true, it calls `push_head()`.
2. Otherwise, it loops until the current node is the node before the index desired.
3. Then it creates a `newNode` containing the data we want.
4. It makes the new node's next pointer point to the same node the current node's next pointer points to, then it makes the current node point to the new node.

### `remove_by_index`

```c
// Code Snippet 11

int remove_by_index(Node **head, int idx) {
	int counter = 0;
	int returnValue = 0;
	Node *current = *head;
	Node *deleteMe = nullptr;
	
	if( idx <0 ) {
		fprintf(stderr, "Can not remove from an index out of range!\n");
	} else if ( idx == 0 )  {
		pop_head(head);
		returnValue = 1;
	} else if ( current == nullptr ) {
		fprintf(stderr, "Can not remove from an empty list!\n");
	} else {
		while( current != nullptr && counter<idx-1 ) {
			current = current->next;
			++counter;
		}
		
		if( current == nullptr || current->next == nullptr ) {
			fprintf(stderr, "Can not remove from an index out of range!\n");
		} else {
			deleteMe = current->next;
			current->next = deleteMe->next;
			free(deleteMe);
			returnValue = 1;
		}
	}
		
	return returnValue;
}

int main(void) {
	Node* myList = nullptr;
	push_head(&myList, 10, 2);
	push_head(&myList, 12, 2);
	push_head(&myList, -9, 2);
	
	if(remove_by_index(&myList, 2)) {
		printf("Removed Successfully\n");
	}
	if(remove_by_index(&myList, 0)) {
		printf("Removed Successfully\n");
	}
	
	return 0;
}
```

Before anything, it checks if the index is not negative. If true:

1. It checks if the index is zero, then it calls `pop_head()`.
2. Then it checks if the list is not empty. If true, it loops until the current node is the node before the one to be deleted.
3. Then it makes the current node's next pointer point to the node after the node to be deleted.
4. Then it deallocates the node to be deleted.

## Types of Linked Lists

### 1. Singly Linked List

It begins with a pointer to the first node called head, and terminates with a null pointer. It is only traversed in one direction.

### 2. Circular Singly Linked List

Its last node's next pointer points to the first node, making the traversal go infinite.

### 3. Doubly Linked List

Each node has a `next` and `prev` pointer. It allows traversal in both directions.

### 4. Circular Doubly Linked List

The next pointer of the last node points to the first node, and the prev pointer of the first node points to the last node.

---

You can have a pointer to the tail, and it can save you some complexity in time, but it will add complexity in space.

## Which implementation to choose?

It depends on your application — do you need time, space, or a compromise between the two?
