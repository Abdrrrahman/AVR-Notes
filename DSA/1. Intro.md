# Introduction to Data Structures in C

This part is documentation for the data structures session in C. We are covering only the linear data structures. A Data Structure is a collection of data stored in memory in a certain way and accessed through a certain method. Each data structure has its own way of accessing data and storing it.

Array is one of the basic data structures known in C. It is stored in memory in a contiguous way, and it is accessed in a random way (random access).

We have two implementations for each data structure: one is the array implementation, and the other is the linked list implementation. The Array Stack implementation differs from the normal array in accessing memory, but the storing is the same. The same rule applies to the Linked Stack.

I am not going to explain the data structures logic from scratch — I am just migrating from data structures in C++ to data structures in C.

A quick note: in all implementations I use `nullptr` instead of `NULL`. `nullptr` was introduced to C in C23; it is not a legacy option, so most codebases and implementations will still use `NULL` instead of `nullptr`, although `nullptr` is safer.

We will use generics with `void *`, and also generic print functions.

Since `genericPrint.h` gets included by more than one `.c` file (like `LinkedList.c`, and later other data structure files), the function itself can't live in the header as a full definition — `#pragma once` only stops a header from being included twice within the _same_ translation unit, not across multiple `.c` files. If the full function body sat in the header, every `.c` file that includes it would produce its own copy, and the linker would fail with a "multiple definition" error the moment two of them get linked together.

So, matching the same pattern used for `LinkedList.h`/`LinkedList.c`, the declaration goes in the header and the definition goes in its own `.c` file, this also keeps a single copy of the function in flash, which matters for embedded targets.

We will use generics with `void *`, and also a generic print function that can print multiple primitive data types.

Since `genericPrint.h` gets included by more than one `.c` file (such as `LinkedList.c`, and later the implementations of other data structures), the function body cannot live inside the header. `#pragma once` only prevents a header from being included multiple times within the **same** translation unit, it does **not** prevent multiple `.c` files from generating their own copy of a function defined inside that header.

Therefore, we follow the same pattern as the rest of the project:
- `genericPrint.h` contains the datatype definitions and the function declaration.
- `genericPrint.c` contains the implementation.

This keeps only one copy of the function in the final executable and avoids linker errors.
## genericPrint.h
```c
#pragma once

#include <stdio.h>

typedef enum {
    TYPE_BYTE,
    TYPE_SHORT,
    TYPE_INT,
    TYPE_LONG,
    TYPE_LONG_LONG,
    TYPE_FLOAT,
    TYPE_DOUBLE,
    TYPE_LONGDOUBLE,
    TYPE_CHAR,
    TYPE_STRING
} Datatype;

void genericPrint(const void *data, Datatype type);
```

## genericPrint.c
```c
#include "genericPrint.h"

void genericPrint(const void *data, Datatype type) {
    if (data == nullptr) {
        printf("(null)\n");
        return;
    }
	
    switch (type) {
        case TYPE_BYTE:
            // Printed as a hexadecimal byte
            printf("0x%02X\n", *(const unsigned char *)data);
            break;
			
        case TYPE_SHORT:
            printf("%d\n", *(const short *)data);
            break;
			
        case TYPE_INT:
            printf("%d\n", *(const int *)data);
            break;
			
        case TYPE_LONG:
            printf("%ld\n", *(const long *)data);
            break;
			
        case TYPE_LONG_LONG:
            printf("%lld\n", *(const long long *)data);
            break;
			
        case TYPE_FLOAT:
            printf("%f\n", *(const float *)data);
            break;
			
        case TYPE_DOUBLE:
            printf("%lf\n", *(const double *)data);
            break;
			
        case TYPE_LONGDOUBLE:
            printf("%Lf\n", *(const long double *)data);
            break;
			
        case TYPE_CHAR:
            printf("%c\n", *(const char *)data);
            break;
			
        case TYPE_STRING:
            // data itself is the pointer to the first character
            printf("%s\n", (const char *)data);
            break;
			
        default:
            fprintf(stderr, "Error: Unknown data type token.\n");
            break;
    }
}
```
